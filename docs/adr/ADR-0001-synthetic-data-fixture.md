# ADR-0001 · Synthetic-data fixture vs PPMI-only access

- Status: **Accepted**
- Date: 2026-05-20
- Deciders: Pacheco-Barrios; Rolston Lab

## Context

PPMI's Data Use Agreement prohibits redistribution of patient-level data.
A public research-code repository accompanying a paper must therefore
either (a) be unrunnable without first requesting PPMI access — typically
a one-week turnaround — or (b) ship a fake-data fixture that lets
reviewers and replicators exercise the pipeline immediately.

A repository that cannot be exercised cannot be peer-reviewed by anyone
who has not already applied to PPMI. That single fact dwarfs every other
consideration about reproducibility, citability, and pedagogical value.

## Decision

Ship a **deterministic, seeded synthetic PPMI fixture** under
`data-synth/`, generated by `R/helpers/make_synthetic_cohort.R` with
`set.seed(20260519)`. The fixture preserves variable names, ordinal
ranges (NP1PAIN 0–4, NHY 1–5, etc.), arm proportions (105 DBS / 1,379
Never-DBS), and rough marginal distributions. It does **not** preserve
joint distributions, temporal correlations, or any real treatment
effect. `data-synth/README.md` labels it explicitly as FAKE DATA — DO
NOT USE FOR INFERENCE.

The repo defaults to synthetic mode (`config.example.yml` sets
`use_synth: true`). Real PPMI data is opt-in via a local `config.yml`
(gitignored) that flips `use_synth: false` and supplies
`ppmi_data_root`.

## Consequences

**Positive**
- A reviewer with no PPMI access can clone the repo, run `make all`,
  and verify the pipeline end-to-end in under 20 minutes.
- CI on GitHub Actions can run integration tests on every push.
- The synthetic data forms a regression-test substrate for the whole
  pipeline (Sprint outputs reproducible from seed).
- Pedagogical value for trainees outside the lab.

**Negative**
- Synthetic numbers cannot reproduce manuscript figures; this must be
  emphasized in `README.md` and `REPRODUCE.md` to prevent
  misinterpretation.
- Maintenance burden: when the real schema evolves (e.g., new PPMI
  cut), the synthetic generator must be updated in lock-step.

## Alternatives considered

- **Aggregated-only outputs (no fixture).** Reviewers cannot rerun
  anything. Rejected.
- **Real-data-only, PPMI gate.** Imposes a 1-week turnaround on every
  reviewer. Rejected as too friction-heavy.
- **Bundle a thin PPMI subset behind a password.** Violates the DUA.
  Rejected.
